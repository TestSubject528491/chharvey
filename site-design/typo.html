Paragraph Scheme

Paragraphs are the backbone of a document; they are made to organize and group textual ideas together. In the written media, paragraphs can span anywhere from 3 to 8 sentences. However on the Web, they can be much much shorter.

This is a lede paragraph. A lede is traditionally an introductory or summary sentence or paragraph to catch the reader's attention. If a lede sentence is part of a paragraph, it acts like a mini-heading for that text. This site's ledes are designed to catch the reader's eye, as they are bolder, darker, more spaced, and in a sans-serif font, which has a slightly slower reading speed than serif fonts. Use the class .lede on whole paragraphs, or on the first span within a paragraph as below.

This is a paragraph in prose. Prose paragraphs are offset from the page because they are a bit more formal and provide more detail. Rather than a typical box or list users can skim, they usually require more careful reading. It is for this reason that paragraphs in prose are presented in a serif font, which is believed to help the reader's eyes move faster along the page. Prose, being more formal than other web-based texts, is left-right justified. The .prose class is used for this style.

A caption describes in more detail the contents of the image, figure, example, or table. Captions may also include the title of the figure in a .lede span. It is styled with smaller, sans-serif text, and is usually shorter and more concise. The .caption class can be applied to a paragraph, table caption, or figure caption.

Though rare, captions may contain multiple paragraphs. They should not be styled in prose.

Usually, this is not the case since a caption is typically too short to span multiple paragraphs.

Block quotes are material taken directly from another source. They are usually very short and styled larger than the default font. Use the <blockquote> element, or use the .q-block class on a paragraph or div. Source
It's courteous (and probably required) to reference the source of the quoted text. Apply the .source class to a span at the end of the block quote. Source

Block quotes that are longer than about 40 words should contain paragraph(s) in prose. This is because longer passages of text are more easily read in a serif font. Apply the .prose class to the entire block quote instead of to each paragraph inside.

The .source should be its own paragraph. Include the year as well. If you're really picky, you can wrap the year in a <time> tag.

  1. This is a prose list (ol.prose). A prose list is an ordered list whose items consist of flow-content. Each list item contains one or more paragraphs. Since the .prose class is applied to the ordered list, it does not need to apply to the paragraphs within the list.

  2. Typically, each list item in a prose list will have a lede. The first sentence of each list item can be thought of as a lede or heading (use the .lede class), although it does not take part in the DOM tree.

    The list-style-type of a prose list is not necessarily specified, although it is conventional to use decimal.

  3. Sometimes, a prose list item may have a lede that acts like a heading.

    In this case, make it a separate paragraph. Again, this lede does not affect the DOM tree. Actual headings (h1, etc.) are permissible in list items, but they usually make the DOM tree look icky.

Sometimes, examples are useful for demonstrating or illustrating something. Examples are styled specially to draw the reader's attention.

Example paragraphs are best not written in prose.

Table Scheme

This is a list-style table.

List-style tables are best for displaying information that correlates across rows.

In a list-style table, a table header would be a row or row-group of cells. If a table header is present, its cells may be normal cells or heading cells (headings). The headings, if present, should have a scope of col. Similarly, these headings should be targeted by all the cells in its column.

An example of a list-style table might be a list of student information including names, ages, genders, and ID numbers.

This is a bar-style table.

Bar-style tables are best for displaying information that correlates down columns.

In a bar-style table, a table header would be a column or column group of cells. If a table header is present, its cells may be normal cells or heading cells (headings). The headings, if present, should have a scope of row. Similarly, these headings should be targeted by all the cells in its row.

An example of a bar-style table might be a spreadsheet displaying the position, velocity, and acceleration values of a falling object across time.

This is a grid-style table.

Grid-style tables are best for displaying information that either correlates across rows, down columns, or tabular data that correlates among both rows and columns.

In a grid-style table, a table header could be a row or row group of cells, or a column or column group of cells. If table header(s) are present, their cells may be normal cells or heading cells (headings). The headings, if present, can have a scope of either row or col. Similarly, these headings can be targeted by any cells.
