The Grid

This page describes the grid system of the site. The grid implements the structure and layout of any page. It consists of rows and cells, treated as table rows and table cells respectively.

The classes used in this grid system determine where elements of a page are positioned. Therefore, the classnames are abstract. Since the grid consists of containers, and not actual content, the classnames are acceptable. See Nicole Sullivan's view on separation of container with content.

Seeing as the grid is presentational, it should degrade gracefully. This means that if someone were to “turn off” the grid, you would still be able to read and interact with the website without any problems. It might not look pretty, but the content should be intact.

Setting It Up

To set up a grid, you'll need to create a row with cells inside. The effect will be a table-like structure.

Custom Cells

Each row is 100% the width of its container, unless otherwise specified. Each row can have any number of cells. Notice that in the example above, that the cells were not evenly distributed: the middle cell will likely be significantly wider than the other two cells, assuming there will be more text in that element. If you want the row to force the cells to have equal widths, use cell subclasses to specify their widths.

Nested Rows

There is something else worth mentioning. It is possible to have more than 5 cells in a row by nesting rows. This means putting rows within rows. More accurately, rows within cells within rows.

For example, lets say you want twelve equal-width cells. Then you could create a row with three .Gcol.F1o3s, and in each cell, build a row with 4 .Gcol.F1o4s each. You could also have four groups of three, two groups of three groups of two, etc.
