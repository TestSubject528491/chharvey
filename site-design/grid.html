Site Design — The Grid

Updated Jan 23, 2014.

This page describes the grid system of the site. The grid implements the structure and layout of any page. It consists of lines and units, treated as table rows and table cells respectively.

Important 2014-01-23: Since the classes used in the grid system define where the elements are positioned, the classnames are purely presentational and non-semantic. Therefore, the developer is encouraged to use concrete semantic classnames where relevant, and extend the classes in grid.less.

Seeing as the grid is presentational, it should degrade gracefully. This means that if someone were to “turn off” the grid, you would still be able to read and interact with the website without any problems. It might not look pretty, but the content should be intact.

Setting It Up

To set up a grid, you'll need to create a line with units inside. The effect will be a table-like structure, with the line acting as a row and the units acting as cells.

Of course, a good developer wouldn't use the .line and .unit classes themselves, but would extend them to more meaningful classes. This way, the HTML author wouldn't have to worry about how the elements are styled, and in the future if the CSS designer wants to change the way the elements look, they can just go into the CSS and change some styles rather than having to change the classnames in the HTML.

Custom Units

Each line is 100% the width of its container, unless otherwise specified. Each line can have any number of units. Notice that in the example above, that the units were not evenly distributed: the middle unit will likely be significantly wider than the other two units, assuming there will be more text in that element. If you want the line to force the units to have equal widths, you must add classes to the units to specify their widths. See the following classes:

Note that these classes automatically extend the .unit class in grid.less.

Nested Lines

There is something else worth mentioning. It is possible to have more than 6 units in a line by nesting lines. This means putting lines within lines. More accurately, lines within units within lines.

For example, lets say you want twelve equal-width units. Then you could create a line with two .unit-1to2s, and in each unit, build a line with 6 .unit-1to6s each. You could also have three groups of four, etc.
