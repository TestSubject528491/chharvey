The Grid

This page describes the grid system of the site. The grid implements the structure and layout of any page. It consists of rows and cells, treated as table rows and table cells respectively.

Important 2014-01-23: Since the classes used in the grid system define where the elements are positioned, the classnames are purely presentational and “unsemantic.” This is okay, however, since the grid consists of mere containers, not actual content itself. The classnames don't describe content because the contents within these containers are expected to do so. An alternative is to use concrete semantic classnames where relevant, and extend the classes in grid.less.

Seeing as the grid is presentational, it should degrade gracefully. This means that if someone were to “turn off” the grid, you would still be able to read and interact with the website without any problems. It might not look pretty, but the content should be intact.

Setting It Up

To set up a grid, you'll need to create a row with cells inside. The effect will be a table-like structure.

Of course, a good developer wouldn't use the .row and .cell classes themselves, but would extend them to more meaningful classes. This way, the HTML author wouldn't have to worry about how the elements are styled, and in the future if the CSS designer wants to change the way the elements look, they can just go into the CSS and change some styles rather than having to change the classnames in the HTML.

Custom Cells

Each row is 100% the width of its container, unless otherwise specified. Each row can have any number of cells. Notice that in the example above, that the cells were not evenly distributed: the middle cell will likely be significantly wider than the other two cells, assuming there will be more text in that element. If you want the row to force the cells to have equal widths, there are two methods.

Custom Row Names

The first method is to apply a subclass to the row. Instead of calling the row .row, think about calling it .row--trio, if it has three cells. This subclass will force all three cells to have equal widths: 33.333%

The table below lists custom subclasses of .row.

.row--solo > .cell
width: 100%;
.row--duet > .cell
width: 50%;
.row--trio > .cell
width: 33.333%;
.row--quartet > .cell
width: 25%;
.row--quintet > .cell
width: 20%;
.row--sextet > .cell
width: 16.666%;

Nested Rows

There is something else worth mentioning. It is possible to have more than 6 cells in a row by nesting rows. This means putting rows within rows. More accurately, rows within cells within rows.

For example, lets say you want twelve equal-width cells. Then you could create a row with two .cell--1o2s, and in each cell, build a row with 6 .cell--1o6s each. You could also have three groups of four, etc.
