The Grid

This page describes the grid system of the site. The grid implements the structure and layout of any page. It consists of rows and cells, treated as table rows and table cells respectively.

Important 2014-01-23: Since the classes used in the grid system define where the elements are positioned, the classnames are purely presentational and non-semantic. Therefore, the developer is encouraged to use concrete semantic classnames where relevant, and extend the classes in grid.less.

Seeing as the grid is presentational, it should degrade gracefully. This means that if someone were to “turn off” the grid, you would still be able to read and interact with the website without any problems. It might not look pretty, but the content should be intact.

Setting It Up

To set up a grid, you'll need to create a row with cells inside. The effect will be a table-like structure.

Of course, a good developer wouldn't use the .row and .cell classes themselves, but would extend them to more meaningful classes. This way, the HTML author wouldn't have to worry about how the elements are styled, and in the future if the CSS designer wants to change the way the elements look, they can just go into the CSS and change some styles rather than having to change the classnames in the HTML.

Custom Cells

Each row is 100% the width of its container, unless otherwise specified. Each row can have any number of cells. Notice that in the example above, that the cells were not evenly distributed: the middle cell will likely be significantly wider than the other two cells, assuming there will be more text in that element. If you want the row to force the cells to have equal widths, you must add classes to the cells to specify their widths. See the following classes:

Note that these classes automatically extend the .cell class in grid.less.

Custom Block Widths

Sometimes you may want to change the widths of a block or a .row. For this, use classes for custom widths.

The following mixins change the appearance of general block elements. They can be included in the class of any block element, or to any extension of .rows, which contain .cells.

Caution: For the sake of simplicity, I'll be using the Less mixin names for the HTML elements' class attributes; I am merely saving time and space. As the developer, you are encouraged to give the elements semantic classnames and include the Less mixins.

.linebreadth-full

A normal block element defaults to 100% width and no left/right margins. This is equivalent to the .linebreadth-full class, which you can use to override another class if need be.

Nested Rows

There is something else worth mentioning. It is possible to have more than 6 cells in a row by nesting rows. This means putting rows within rows. More accurately, rows within cells within rows.

For example, lets say you want twelve equal-width cells. Then you could create a row with two .cell--1o2s, and in each cell, build a row with 6 .cell--1o6s each. You could also have three groups of four, etc.
