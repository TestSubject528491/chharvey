What kind of heading is it?

The structure and style of nested headings in Google Chrome

This page examines how an HTML outliner will parse multiple types of headings in multiple nested sections. For the sake of consistency, use the following numberings for headings, according to nest level.

"BODY"
upper-Roman (I, II, III, ...)
upper-alpha (A, B, C, ...)
decimal (1, 2, 3, ...)
lower-alpha (a, b, c, ...)
lower-Roman (i, ii, iii, ...)
child section
an explicit direct descendent of an element
subsection
implicit descendent created by the browser's parsing rules
sister section
explicit element that shares the same parent
"co-section"
implicit sister section
"behaves like"
how the browser parses the headings and organizes them into an accurate DOM

BODY.
This is a h1 in the `body` section
"behaves" like a h1
styled like h1

Although the HTML5 spec does not state that the body element is a sectioning content element, it really is. The body element is the root of the DOM tree, therefore it is the root section. I have learned from experience not to use the article section (because it is sectioning content) as the only direct descendent (a.k.a. child) of the body element. It creates an unnecessary subsection. The article element is meant for use when multiple sub-sections (namely, articles) of the body are present.
