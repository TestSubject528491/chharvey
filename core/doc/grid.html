The Grid

This page describes the grid system of the site. The grid implements the layout of any page. It consists of rows and cells, treated as table rows and table cells respectively.

The classes used in this grid system determine where elements of a page are positioned. Therefore, the classnames are intentionally “unsemanic.” Since the grid consists of containers, and not actual content, the classnames are acceptable. See Nicole Sullivan’s view on separation of container with content.

Think of the grid as shelves. The shelves are set up first, and then the contents are placed on them. The rows and columns of the grid have the respective classes .Grow and .Gcol.

Seeing as the grid is presentational, it should degrade gracefully. This means that if someone were to “turn off” the grid, you would still be able to read and interact with the website without any problems. It might not look pretty, but the content should be intact.

Setting It Up

To set up a grid, you’ll need to create a row with cells inside. The effect will be a table-like structure.

Custom Cells

Each row is 100% the width of its container, unless otherwise specified. Each row can have any number of cells. Nested Rows

There is something else worth mentioning. It is possible to have more than 5 cells in a row by nesting rows. This means putting rows within rows. More accurately, rows within cells within rows.

For example, lets say you want twelve equal-width cells. Then you could create a row with three .Gcol.F1o3s, and in each cell, build a row with 4 .Gcol.F1o4s each. You could also have four groups of three, two groups of three groups of two, etc.

Notice how in these examples there were rows within cells. If you are nesting rows, you must contain the second-level row within the first-level cell. The .Grow and .Gcol classes were not meant to mix in the same element. There shouldn’t be any elements with both classes in your HTML.
