The Grid

This document will eventually be converted into a blog post and relocated under /blog/grid.html. For version control reasons, I am keeping it under /core/doc/ with the other style guide documentation until it, along with the new grid system (#20), is finished.

There are many grid systems out there. This article does not go into any of them. Rather, after reading this article, hopefully you will see the anti-need for any grid system, and certainly how ridiculous it is that there are so many of them. You will create your own grids using pure CSS. No downloads, no pre-defined classes, and definitely no unsemantic markup.

Before delving too far, though, you should probably get yourself up-to-speed on the CSS Flexible Box Layout Module, or Flexbox for short.

This page describes the grid system of the site. The grid implements the layout of any page. It consists of rows and cells, treated as table rows and table cells respectively.

The classes used in this grid system determine where elements of a page are positioned. Therefore, the classnames are intentionally unsemanic. Since the grid consists of containers, and not actual content, the classnames are acceptable. See Nicole Sullivan’s view on separation of container with content.

Think of the grid as shelves. The shelves are set up first, and then the contents are placed on them. The rows and columns of the grid have the respective classes .Grow and .Gcol.

Seeing as the grid is presentational, it should degrade gracefully. This means that if someone were to turn off the grid, you would still be able to read and interact with the website without any problems. It might not look pretty, but the content should be intact.

Setting It Up

To set up a grid, you’ll need to create a row with cells inside. The effect will be a table-like structure.

div class=Grow this is a row! div class=Gcolthis is col 1!div div class=Gcolthis is col 2!div div class=Gcolthis is col 3!div div
this is col 1!
this is col 2!
this is col 3!

Outlines are added to show structure, and widths are not equal to show they are not fixed.

Custom Cells

Each row is 100% the width of its container, unless otherwise specified. Each row can have any number of cells. Notice that in the example above, that the cells were not evenly distributed: the middle cell will likely be significantly wider than the other two cells, assuming there will be more text in that element. If you want the row to force the cells to have equal widths, use cell subclasses to specify their widths.
