Lists and Prose

This section describes common ways of combining lists with normal prose. The first method is used for listing whole blocks of content, for example a description after each list item. The second method is for using lists within prose.

A List of Prose, or a LOP, is simply a .List.O Object whose child .Items contain .P Objects.

  1. A LOP is a .List.O whose .Items consist of prose. Each .Item contains only .P Objects.

  2. Typically, each list item in a prose list will have a lede sentence. The lede sentence is treated like an actual list item in a short list, if the rest of the prose were not present. It can also be be thought of as a lede or heading for the text, although it does not take part in the Document Outline. The lede sentence must be a .Lede Object at the beginning of the .Item.

    Though literal h1 Elements are technically permissible inside li Elements, they usually make the Document Outline look icky, so we don’t use them.

Combining p and ol Elements, we can create what is known as a List in Prose, or a LIP. Three things happen with this construction:

  1. we use three Elements: a p, an ol.Enum.LIP, and another p, all representing inline spans of text, contained within a div.P, which represents the paragraph Object;
  2. each ol.Enum.LIP > li.Item Object also appears inline, as if it were part of the paragraph itself; and
  3. the containing Paragraph Object, represented by the div.P, is still displayed as a block so that its surrounding siblings do not appear on the same line.

With these three taking place, one can utilize the semantics of the ordered list while still maintaining the style of continuous prose. An .Enum.LIP differs from a normal .Enum in that the items are numbered.

These methods provide for a more semantic Web. The source code is cleaner, the Elements provide semantics and formatting, and some of the CSS syntax (i.e., delimiters) are automated. However, it can only be used for CSS, not Less, syntax. That means no commands, arguments, or nested selectors. Another disadvantage is that the markup cannot be contained in a code Element: by HTML standards, code can only contain phrasing-level content, however by these standards, a .Codeblock.CSS must contain dl(s).

Other Technical Text: Inputs and Outputs

A .Key object represents a physical key (or button) on a computer keyboard, calculator, or other electronic device. It may be instantiated by a kbd > kbd Element.

The outer kbd represents a sequence of user input, while the inner kbd represents an actual key to be pressed.

A .Dir Object is used to represent a path in a file directory, or a URL or URI. It may be instantiated by a code Element or a samp Element.
