Notice the differences between the title and subtitle. The title is in a sans-serif, monoweight type, in bold and small caps, which provides a very rigid and blocky look. Code Blocks
The pre Element, by itself (without a child code Element),
will be used for blocks of computer code.

function fibonacci(n) {
  if (n === 0 || n === 1) return 1;
  else if (n > 1)         return fibonacci(n-1) + fibonacci(n-2);
  else                    return NaN;
}

Warning: As of 2015-04-30 there is no longer a mechanism available for automated syntax highlighting in code blocks. Manual highlighting is described below. Check back later for updates.

The Elements pre > small and pre > strong respectively denote code comments (an actual thing), and importance. (Importance isn’t an actual thing in code, since all code is important.)

function fibonacci(n) {
  if (n === 0 || n === 1) return 1; // these are the base cases
  else if (n > 1)         return fibonacci(n-1) + fibonacci(n-2);
  else                    return NaN; // a strong denotes importance here,
                                      // which highlights the case of inputting
                                      // a negative number
}
/* Comment punctuation (/*, */, and //) is not included; must type manually. */

/* … otherwise, it would look like this, which is distracting. */

Tables

tl;dr: Add class .Table to a table Element to make it appear as an encapsulated unit (like a paragraph). In the .Table, use .Rowgroup on thead, tfoot, and tbody Elements to add the borders above and below. Use subclasses .Head, .Foot, and .Body for specific display properties. Use .Row on tr Elements to add odd and even (or head) coloring. Use .Cell on all th and td Elements to add padding and min-height. Use subclass .Hc on th for a heading look (font-family and bold).

Documents

Note: On all .c-Documents in this section, outlines have been added to show structure.

A document may take the form of a body > main or body >> article. The said document should have a status and be marked up with metadata. Use the .c-Document class to style the document, and use modifier subclasses to style the document according to its status if it has one. Notice we are separating semantics with presentation: the [data-status] attribute is unrelated to what class is applied to the Element.
