Typography

This Style Guide demonstrates the appearance of CSS Objects using site-wide styles.

Fonts

The base font-family is used for the main body copy. It is a sans-serif polyweight type with a moderate thick/thin transition. Being a sans-serif, its italic is actually an oblique, which just slants the glyphs. Suggested typefaces include Optima, Lucida Grande, Tahoma.

The prose font-family is used for text in formal documents and articles. It is a serif oldstyle type, which is easy on the eyes and can be read at extensive length. Like the base font, it, too, has a moderate thick/thin transition, but has diagonal stress and serifs, and true italics, which means it uses a whole new set of glyphs. Suggested typefaces include Baskerville, Palatino, Cochin.

The heading font-family is used only for headings of body copy (not table headings, etc.). It is a sans-serif type with little to no thick/thin transition. It is extremely lightweight, which makes a great heading when boldened, but it should not be used for extensive amounts of text. Suggested typefaces include Avenir, Seravek.

The accent font-family is used for labels, block quotes, butons/keys, and other special elements. Like the heading font-family, it is a monoweight sans-serif, yet slightly heavier. Suggested typefaces include Geneva, Tahoma.

The code font-family is a monospace type used for… well, code. Personally, I dislike monospace fonts with serifs, so this family is sans-serif. Suggested typefaces include Menlo, Monaco, Andale Mono.

The math font-family is a serif modern type use for mathematical spans and spans in foreign languages. It has a radical thick/thin transition, vertical stress, and straight horizontal serifs. With a cold yet elegant look, this font family makes long spans of text hard to read, but that’s okay because this font is used only for short expressions. Suggested typefaces include Didot, Bodoni 72, Times.

The script font-family is used for very fancy elements such as drop-caps and subheadings. Use this font family sparingly. Suggested typefaces include Zapfino.

Snell Roundhand is another script typeface used sparingly throughout the site.

Lists and Prose

This section describes common ways of combining lists with normal prose. The first method is used for listing whole blocks of content, for example a description after each list item. The second method is for using lists within prose.

A List of Prose, or a LOP, is simply a .Olist.LOP Object whose child .Items contain .P Objects.

  1. An .Olist.LOP has .Items that consist of prose. Each .Item contains only .P Objects. An .Olist.LOP differs from a normal .Olist in that each non-first item is separated from the previous item.

  2. Typically, each list item in a prose list will have a lede sentence. The lede sentence is treated like an actual list item in a short list, if the rest of the prose were not present. It can also be be thought of as a lede lede or heading for the text, although it does not take part in the Document Outline. The lede sentence must be a .Lede Object at the beginning of the .Item.

    Though literal h1 Elements are technically permissible inside li Elements, they usually make the Document Outline look icky, so we don’t use them.

Combining p and ol Elements, we can create what is known as a List in Prose, or a LIP. Three things happen with this construction:

  1. we use three Elements: a p, an ol.Ilist.LIP, and another p, all representing inline spans of text, contained within a div.P, which represents the paragraph Object;
  2. each ol.Ilist.LIP > li.Item Object also appears inline, as if it were part of the paragraph itself; and
  3. the containing Paragraph Object, represented by the div.P, is still displayed as a block so that its surrounding siblings do not appear on the same line.

With these three taking place, one can utilize the semantics of the ordered list while still maintaining the style of continuous prose. A .Ilist.LIP differs from a normal .Ilist in that the items are numbered.

A .Map Object is similar to a dl Element in that each contains groups of name-value pairs. In fact, the dl Element is best suited as an instance of the .Map class. Each group of a .Map must have exactly 1 .Term and exactly 1 .Desc. Thus a .Map can be easily converted into a JSON object.

Other Technical Text: Inputs and Outputs

A .Key object represents a physical key (or button) on a computer keyboard, calculator, or other electronic device. It may be instantiated by a kbd > kbd Element.

In a kbd > kbd, the outer kbd represents a sequence of user input, while the inner kbd represents an actual key to be pressed.

A .Dir Object is used to represent a path in a file directory, or a URL or URI. It may be instantiated by a code Element or a samp Element.
